# string+

Реализация стандартной библиотеки `string.h` на языке C с дополнительными функциями обработки строк.  
Этот проект является учебной работой, направленной на углубленное понимание работы со строками, памятью и форматированным выводом.

## 💡 О проекте

В рамках данного проекта была создана собственная версия популярной библиотеки `string.h`, а также реализованы функции `sprintf` (частично) и набор дополнительных утилит для работы со строками.

**Цель проекта:** Изучение внутреннего устройства строк, управление памятью, парсинг форматных строк и написание качественного кода на C в соответствии со стандартами.

Проект написан на чистом C (стандарт C11) и собран в статическую библиотеку.

## 🛠 Реализованный функционал

### 📚 Часть 1. Стандартная библиотека `string.h`


| Функция | Описание |
|---------|----------|
| `memchr` | Поиск символа в блоке памяти |
| `memcmp` | Сравнение двух блоков памяти |
| `memcpy` | Копирование блоков памяти |
| `memset` | Заполнение блока памяти заданным значением |
| `strncat` | Конкатенация двух строк с ограничением |
| `strchr` | Поиск первого вхождения символа в строку |
| `strncmp` | Сравнение двух строк с ограничением |
| `strncpy` | Копирование строки с ограничением |
| `strcspn` | Поиск первого совпадения из набора символов |
| `strerror` | Получение строки сообщения об ошибке по её номеру |
| `strlen` | Вычисление длины строки |
| `strpbrk` | Поиск первого вхождения любого символа из набора |
| `strrchr` | Поиск последнего вхождения символа в строку |
| `strstr` | Поиск подстроки |
| `strtok` | Разделение строки на токены |

Дополнительно:
- Введен собственный тип `s21_size_t`.
- Определен макрос `S21_NULL`.
- Реализованы массивы ошибок для Linux и macOS (платформозависимость).

### 🖨 Часть 2. Частичная реализация `sprintf`

Функция `int s21_sprintf(char *str, const char *format, ...);` поддерживает следующие возможности форматирования:

- **Спецификаторы:** `%c`, `%d`, `%f`, `%s`, `%u`, `%%`.
- **Флаги:** `-` (выравнивание влево), `+` (знак числа), пробел.
- **Ширина:** задается числом.
- **Точность:** задается через точку и число (`.число`).
- **Модификаторы длины:** `h` (короткое целое), `l` (длинное целое/двойное).

### ✨ Часть 3. Дополнительные функции обработки строк

Набор полезных функций, расширяющих стандартную библиотеку.

| Функция | Описание |
|---------|----------|
| `s21_to_upper` | Перевод всех символов строки в верхний регистр. |
| `s21_to_lower` | Перевод всех символов строки в нижний регистр. |
| `s21_insert`   | Вставка строки `str` в строку `src` начиная с индекса `start_index`. |
| `s21_trim`     | Удаление указанных символов `trim_chars` в начале и конце строки `src`. |

> **Примечание:** В случае ошибки (нехватка памяти, некорректные аргументы) функции возвращают `S21_NULL`.

## ⚙️ Сборка и установка

Проект собирается в статическую библиотеку `s21_string.a`.

**Основные команды Makefile:**

```bash
make all        # Сборка библиотеки
make clean      # Очистка временных и объектных файлов
make test       # Компиляция и запуск unit-тестов
make gcov_report # Генерация HTML-отчета о покрытии кода тестами
```
Для работы с тестами необходима установленная библиотека `check`.

## 🧪 Тестирование

* **Фреймворк:** `check.h`.
* Для каждой функции написаны модульные тесты, проверяющие её поведение в штатных и граничных ситуациях.
* Результаты тестирования сравниваются с поведением эталонной библиотеки `string.h`.
* Покрытие кода тестами контролируется с помощью `gcov` и должно составлять **не менее 80%**. Отчет генерируется в формате HTML.
